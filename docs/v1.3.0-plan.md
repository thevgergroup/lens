## Goals

The primary gap is **Grok/Aurora on X (Twitter)**: images generated by xAI's Aurora model and shared via `pbs.twimg.com` score near zero because X's CDN strips all metadata and our L3 signals were calibrated on diffusion models, not autoregressive models.

3. Broader social CDN awareness so pixel analysis is always attempted when metadata is unavailable

## Background: Why Aurora Is Different

Aurora (xAI, announced December 2024) is a **next-token autoregressive** model — it predicts image tokens sequentially like a language model, not via iterative diffusion. This matters for detection:

| Property | Diffusion (DALL-E 3, Stable Diffusion, Flux) | Autoregressive (Aurora) |
|----------|----------------------------------------------|------------------------|
| Generation direction | Iterative denoising from pure noise | Left-to-right / raster token prediction |
| Decoder | Latent diffusion VAE | VQ-VAE (vector-quantised) |
| Known artifacts | LSB structure, gradient smoothness, SynthID watermark | VQ-VAE block boundaries, raster-order brightness gradient, high-frequency deficit |
| Metadata at source | EXIF Software, XMP, C2PA | C2PA `trainedAlgorithmicMedia`, EXIF Software "xAI" |
| Metadata after X/Twitter CDN | **Stripped** | **Stripped** |

The key takeaway: after CDN processing, Aurora images must be identified entirely by pixel-domain statistics. No metadata survives.

## What We Know About Aurora's Pixel Fingerprint

### Already implemented (v1.2.0)
- **Inter-block noise correlation (L3):** Aurora scores `hcf` 0.50–0.70 vs. 0.18–0.46 for real photos. Reliable for direct downloads; CDN recompression degrades this signal.
- **R-B channel correlation (L3):** AI images show |corrRB| > 0.95 more often than photos.
- **Gradient smoothness (L3):** Aurora backgrounds are smoother than real photo backgrounds.

### Not yet implemented — candidate signals for v1.3.0

**NPR (Normalized Patch Residuals)**
Compute the difference between each pixel and the mean of its local 3×3 neighbourhood (i.e. a high-pass residual). In real photographs, this residual is dominated by sensor noise — spatially white and channel-independent. In AI images produced by VQ-VAE decoders (Aurora, Stable Diffusion), the residual is structured: flat regions have unnaturally low residual variance, and edges have unnaturally sharp residual spikes. The ratio `max_local_residual_variance / min_local_residual_variance` should be substantially higher in AI images.

This is implementable in O(n) without any library and runs in < 5ms.

**VQ-VAE periodicity**
Aurora's VQ-VAE tokenises the image into patches (likely 8×8 or 16×16 pixels). The decoder reconstructs each patch semi-independently, leaving a weak periodic signal at the patch grid frequency in the FFT magnitude spectrum. This is distinct from JPEG 8×8 blocking (which we can already identify) — it appears at the VQ patch scale and is present in both JPEG and PNG exports.

In the L4 FFT layer: after identifying the JPEG 8×8 frequency, check for additional periodic peaks at multiples of 1/16 of the Nyquist frequency.

**Raster-order luminance gradient**
Autoregressive models generate tokens in a fixed order (typically row by row). Error accumulates across the generation direction due to exposure bias — early tokens condition later ones, and small errors compound. This produces a subtle brightness or detail gradient across the image (typically slightly sharper/brighter at the start of raster order, softer at the end). Measurable as mean luminance or gradient variance in horizontal bands.

This signal is weak and model-specific, and we don't yet know Aurora's exact raster order. Treat as exploratory.

## L3 Changes

### NPR signal (new)

```
1. For each pixel (skip border), compute residual = pixel_value - mean(3×3 neighbourhood)
2. Divide image into 16×16 blocks
3. For each block, compute residual_variance
4. flat_blocks = blocks where luminance_variance < threshold (smooth regions)
5. If len(flat_blocks) < 20, skip (image too textured for this signal)
6. npr_ratio = std(residual_variance of flat_blocks) / mean(residual_variance of flat_blocks)
   — high ratio means AI (very low floor, sharp spikes at edges)
7. If npr_ratio > 2.5 → weight 0.40
   If npr_ratio > 4.0 → weight 0.50
```

This is architecture-agnostic and should generalise to both Aurora and diffusion models.

### Calibration target

After implementing, validate against the fixture set (see below). Target:
- Aurora (direct): score ≥ 0.55 (Possible AI) — currently scores ~0 on CDN-served images
- Aurora (via pbs.twimg.com): score ≥ 0.45 (Possible AI threshold)
- Real photo (pbs.twimg.com): score ≤ 0.20

## L4 Changes

### VQ-VAE periodicity check (new)

In the existing FFT analysis:

```
1. Compute 2D FFT of luminance channel (already done)
2. After identifying JPEG 8×8 frequency peak (at Nyquist/4):
3. Also look for peaks at Nyquist/8 (16-pixel period) and Nyquist/16 (32-pixel period)
4. A peak at Nyquist/8 that is absent in JPEG-only images → VQ-VAE tokenisation artifact
5. Weight: 0.35 (weak, corroborating signal)
```

This is additive with the SynthID check — both can fire independently.

| Category | Target score | Max acceptable false negative |
|----------|-------------|-------------------------------|

A 30% false negative rate on CDN-stripped Aurora images is acceptable for v1.3.0. The goal is to go from the current ~0% detection rate to meaningful coverage.

## What We Are Not Doing in v1.3.0

- **Flux.1 / SDXL without metadata:** Statistical properties still overlap too much with real photos at browser colour depth. Deferred to v1.4.0 pending better research.
- **ML-based classifier:** Keeps the no-external-requests, no-build-step constraints. Any ML approach would require a WASM model bundle, which changes the extension's trust and size profile significantly.
- **Adversarial stripping resistance:** Images deliberately post-processed to evade detection are out of scope. The goal is the common case.
